# Surfr Fork — Changes vs `open-meteo/maps` upstream

Fork base: `upstream/main` (merge-base `f076847`)

This document summarises every substantive change introduced by this fork
relative to the upstream open-meteo maps project, grouped by concern.

---

## 1. Tile caching & CDN architecture

A full 4-tier Cloudflare cache stack sits on top of the upstream client. All
`.om` and `latest.json` traffic flows through `maps.thesurfr.app/tiles/*`
instead of Open-Meteo's origin directly. See
`docs/adr/0001-caching-architecture.md` for the full decision record.

### 1.1 Cloudflare Pages Function proxy — `functions/`

| File | Purpose |
|---|---|
| `functions/tiles/[[path]].ts` | Catch-all tile proxy. Serves `.om` from R2 keyed by the full runPath; falls back to origin (+ lazy R2 fill via `waitUntil`) when R2 is cold. Serves `latest.json` from R2 with `Cache-Control: no-store`. 404s `meta.json` and `in-progress.json`. `.om` responses carry `Cache-Control: public, max-age=2592000, immutable` + `X-Surfr-Cache-Status`. |
| `functions/tiles/_admin.ts` | `/tiles/_admin` HTML dashboard. Per-domain status pills (OK / STALE / COLD / UNKNOWN), R2 vs upstream `latest.json` comparison, file count + MB, oldest/newest valid time, historical runs column (2 prior runs retained), last cron tick metadata. |
| `functions/tiles/_warmer-trigger.ts` | HTTP entry-point for warming. `?wait=1` blocks, default fire-and-forget via `waitUntil`. Validates `?domain=` against the known list. |
| `functions/lib/warmer.ts` | Core warm pipeline: fetch upstream `latest.json` → diff against R2 → fetch upstream `meta.json` for `valid_times` → warm `.om` files concurrency=4, +72 h horizon cap → single atomic PUT of `latest.json` to R2 → `pruneOldRunFiles` (keep current + 2 prior runs). Emits per-run summary. |
| `functions/lib/domains.ts` | Canonical list of 14 warmed tile domains. |
| `wrangler.toml` | Pages project config; binds `TILE_CACHE` R2 bucket (`surfr-tile-cache`). |

Key properties:
- **Run-path in the URL.** Client URLs include `YYYY/MM/DD/HHmmZ`, derived
  from our R2 `latest.json`. Each run has a unique, immutable URL → stable
  ETag → `Range` requests always return 206 → browsers cache with
  `immutable` and skip revalidation entirely.
- **Single atomic swap.** After a successful warm, one `latest.json` PUT
  flips clients to the new run. Upstream's `latest.json` carries
  `valid_times` + `variables` byte-identically to its per-run `meta.json`,
  so a single pointer file is sufficient.
- **2 historical runs retained.** `pruneOldRunFiles` keeps current + 2
  prior runPaths per domain; older runs deleted. Protects tabs left open
  across a swap — they hold their loaded runPath until reload.
- **`latest.json` never cached client-side.** Served with `Cache-Control:
  no-store`. Prevents a stale pointer from producing runPath URLs for a
  run we've pruned.
- **`meta.json` and `in-progress.json` blocked.** Return 404. Only
  `latest.json` can move the client between runs, and it only advances
  after the warm is complete.
- **Horizon cap:** 72 h max from `reference_time`. Outside-horizon `.om`
  requests fall through to origin and opportunistically fill R2 via
  `waitUntil(warmR2)`.
- **Debug endpoint:** `/tiles/_debug/cache?prefix=…` returns JSON
  inventory + per-domain totals + last cron summary.

### 1.2 Cron Worker — `worker-cron/`

Separate Cloudflare Worker whose only job is to fire the warmer every
5 min. Every `.om` URL includes the runPath, so new runs produce new URL
strings and old-run edge entries age out naturally — the cron has no CF
API dependency (no purge, no re-warm).

| File | Purpose |
|---|---|
| `worker-cron/src/index.ts` | `scheduled()` handler that sequentially hits `/tiles/_warmer-trigger?domain=…&wait=1` for each domain, with a 1.5 s pause between domains. Per-domain sharding gives each warm its own CPU/subrequest budget and caps concurrent upstream load. Also exposes `GET /force?domain=X` to re-run a single domain's warmer with the "already up-to-date" short-circuit disabled (manual recovery). |
| `worker-cron/wrangler.toml` | Cron trigger `*/5 * * * *`. No secrets required. |
| `worker-cron/package.json`, `tsconfig.json`, `package-lock.json` | `surfr-tile-warmer-cron` project scaffolding. |

## 2. Frontend client — proxy routing & metadata

`src/lib/helpers.ts` — `getBaseUri()` hard-codes
`https://maps.thesurfr.app/tiles` for all domains (dev + prod).

`src/lib/url.ts` — `getOMUrl()` and `getNextOmUrls()` build canonical URLs
with the runPath baked in:
`${base}/${fmtModelRun(modelRun)}/${fmtSelectedTime(…)}.om`. Routes every
request (including prev/next-hour prefetches) through `getBaseUri()`.
No `?model_run=` URL-param handling — users can't select a run.

`src/lib/metadata.ts` — single `fetch(latest.json, { cache: 'no-store' })`
on page mount populates the `latest` store; `getMetaData` returns a sorted
clone of `latest`. No `meta.json` or `in-progress.json` fetches.

`src/lib/pop-warm.ts` — per-session PoP warm. Reads `latest.json`, builds
range-request warm URLs with runPath, fires at concurrency 4 for the next
72 h of valid times on domain switch.

`src/lib/components/run-date-label.svelte` — small fixed-position label
(`top: 120px, left: 50%`) showing `Run YYYY-MM-DD HH:MMZ`. Rendered in
both standalone and `?embed=1` modes.

## 3. Embed mode (React Native WebView integration)

`src/lib/rn-bridge.ts` — Bidirectional postMessage bridge active only when
`?embed=1` is in the URL. Out-messages: `ready`, `moveend`,
`availableTimestamps`, `availableVariables`, `timestampChanged`,
`forecastLocationSet`, `referenceTime`, `tileFetch`, `storageEstimate`,
`setTimeReceived`, `setVariableReceived`, `setDomainReceived`,
`mapDataLoading`, `mapIdle`. In-messages: `setCenter`,
`setForecastLocation`, `setVariable`, `setDomain`, `setTime`,
`setTzOffsetSeconds`, `setSpotsConfig`. Every out-message stamps
`t = performance.now()` so the host can reconstruct a diagnostic timeline.

`src/routes/+page.svelte` —
- Reads URL params on mount: `clearCache`, `theme`, `opacity`,
  `tz_offset_seconds`, `spots_endpoint`, `spots_token`.
- Runs an `isEmbedMode()` check; if embed, hides the map popup and
  MapLibre's default attribution control.
- Installs `rnBridgeCleanup` in `onDestroy`.
- Swaps the upstream chrome (Dark/Help/Settings/Clipping/Hillshade
  buttons, `Scale`, full `VariableSelection`, `ClippingPanel`,
  `HelpDialog`, `Dropzone`, `Settings`) for a stripped-down standalone UI:
  `OverlayPills` + `ModelPills` + `TimezoneSelector` + `TimeSelector` +
  `KeyboardHandler`.

`src/lib/map-controls.ts` —
- `setMapControlSettings({ embed })` — early-return before adding
  MapLibre's `NavigationControl` / `GeolocateControl` / `GlobeControl` when
  embedded.
- `getStyle()` uses `tiles.openfreemap.org/styles/{positron|dark}`
  (CORS-open).
- Post-load basemap filter: hides transportation / buildings / landcover /
  landuse / waterway / aeroway / park source-layers; hides sub-national
  admin + micro-place labels; forces country borders white (filter
  excludes maritime boundaries); white flat place labels; adds a
  dedicated `surfr_coastline` line layer tracing ocean-class water
  polygons at interpolated width 0.4–1.4 px.
- Wheel/touch zoom rate set to `1/85`.

## 4. Simplified standalone UI

Three Svelte components that replace the upstream `VariableSelection`:

| File | Purpose |
|---|---|
| `src/lib/components/overlay-pills/overlay-pills.svelte` | Wind / Gusts / Rain pill selector. Maps each overlay to alias variable names, picks the first present in `latest.variables`. Special-cases the GFS domain pair (`ncep_gfs013` wind/rain, `ncep_gfs025` gusts). |
| `src/lib/components/overlay-pills/model-pills.svelte` | 13-model dropdown (MET Nordic, Arome-HD, ICON-D2, KNMI NL/EU, UKV, Arome, GEM HRDPS, HRRR, ICON-EU, ECMWF, ICON, GFS). Auto-flips `ncep_gfs013` ↔ `ncep_gfs025` when the user switches between wind and gust overlays on GFS. |
| `src/lib/components/timezone/TimezoneSelector.svelte` | IANA timezone dropdown using `Intl.supportedValuesOf('timeZone')`. Labels include live UTC offset (DST-aware). Persisted to `displayTimezone` (localStorage) and recomputed on time changes. |

## 5. Display-timezone rework

`src/lib/stores/preferences.ts` — `displayTimezone` (persisted IANA string,
defaults to browser TZ) and `displayTzOffsetSeconds` (writable in seconds,
set by RN embed bridge or derived from IANA name).

`src/lib/time-format.ts` — helpers:
- `startOfUTCDay` / `withUTCTime` — pure UTC variants.
- `toShifted` / `fromShifted` — the "shift by offset" trick so UTC getters
  read target-TZ values while `Date` objects stay UTC internally.
- `startOfDisplayDay`, `withDisplayTime`,
  `formatDisplayDate/Time/DateTime`, `getDisplayDate/Month`.
- `getIanaOffsetSeconds(tz, at)` — DST-aware, uses
  `Intl.DateTimeFormat.formatToParts`.

`src/lib/components/time/time-selector.svelte` uses the display-tz
equivalents throughout. No model-run dropdown, lock button, or prev/next-
run navigation (R2's `latest.json` is the single source of truth for the
current run; users can't pick older runs). No `UTC±HH:MM` badge above the
timeline (redundant with `TimezoneSelector`).

Related: `src/lib/stores/time.ts` has no `modelRunLocked` or `inProgress`
store; `src/lib/stores/keyboard.ts` has no `previousModel` / `nextModel` /
`toggleModelRunLock` / `setLatestModelRun` actions;
`src/lib/components/keyboard/keyboard-handler.svelte` has no `m`, `n`,
`ctrl+←`, `ctrl+→` bindings; `src/lib/components/help/help-dialog.svelte`
has no matching hint rows.

## 6. Color scales & rendering

`src/lib/color-scales/surfr.ts` — Surfr wind palette. RGB anchors from
WindguruService.WIND_COLOR_ANCHORS pushed to max-saturation per band
(blue → cyan → green → lime → yellow → orange → red-orange → pink →
magenta → purple). Alpha floor raised to 0.35 for readability over dark
basemaps. Linearly interpolated to 1-kt resolution (breakpoints in m/s).

`src/lib/stores/om-protocol-settings.ts` —
- Registers `surfrWindScale` against keys `wind`, `wind_speed_10m`,
  `wind_gusts_10m`, `wind_u_component_10m`, `wind_v_component_10m`.
- Wraps `defaultResolveRequest` with a hard override: any variable
  matching `/^wind_(speed|gusts|u_component|v_component)(_|$)/` gets its
  `colorScale` and `intervals` force-replaced with `surfrWindScale` (both
  light + dark mode resolved).
- `postReadCallback`: single-flight `AbortController` for prefetching
  adjacent-hour `.om` files so a lingering prefetch can't block the user's
  next scrub click.

`src/lib/layers.ts` —
- Raster opacity: raw pref in both modes (upstream reduces −10 pp in
  dark — our stripped basemap would otherwise look washed out).
- Arrow color: max alpha capped at 0.4.
- Arrow width: 1.3 / 1.1 / 1.0 / 0.9 / 0.8 / 0.7 across thresholds.
- `resolveBeforeLayer(map, desired)` — resilient layer-id lookup: falls
  back to first `boundary*` / `symbol` layer if the requested id isn't in
  the active style (OpenFreeMap uses different ids than the old
  open-meteo basemap).
- `changeOMfileURL` also updates `rasterManager.setBeforeLayer`.
- `onSlowLoad`: forwards the slow-load toast to the RN host via
  `postMessage({ type: 'slowLoadWarning' })`.

`src/lib/constants.ts` —
- `BEFORE_LAYER_RASTER` / `BEFORE_LAYER_VECTOR` /
  `BEFORE_LAYER_VECTOR_WATER_CLIP` set to `waterway_line_label` (the id
  that exists in OpenFreeMap).
- `DEFAULT_CACHE_BLOCK_SIZE_KB` is 192 (3× fewer range requests per
  viewport render than upstream's 64, less HTTP/2 multiplexing
  contention).

## 7. Surfr spots layer

`src/lib/surfr-spots.ts` — Renders Surfr user's saved surf/kite spots as
cyan circles + white labels above zoom 8. Fetches from
`${endpoint}/spots/bbox?…&limit=100` with bearer token, debounced 500 ms
on moveend. Endpoint + token come from URL params (`spots_endpoint`,
`spots_token`) or RN bridge (`setSpotsConfig`). Cancels in-flight bbox
fetch via `AbortController` on every map move.

---

## 8. Routing

`src/routes/+layout.ts` — `ssr = false`. Disables SSR to avoid
`window` / `navigator` ReferenceErrors during prerender since several
components touch those at module top level (e.g. `time-selector.svelte`).

## Summary of behavioural differences vs upstream

| Concern | Upstream | Surfr fork |
|---|---|---|
| Tile origin | Open-Meteo's CDN direct | CF Pages Function proxy + R2 + 5-min cron warmer |
| Client URL shape | Full `YYYY/MM/DD/HHmmZ/<validTime>.om`, runPath derived from a client-held `meta.json` that can drift mid-session | Same shape, runPath derived from our R2 `latest.json` fetched fresh on every page load |
| Pointer files | `latest.json` + per-run `meta.json` + `in-progress.json` | `latest.json` only; `meta.json` and `in-progress.json` 404'd |
| `latest.json` caching | Upstream: no `Cache-Control` (browser heuristic) | `Cache-Control: no-store`; always fresh from R2 |
| `.om` caching | Browser cache on full URL | `Cache-Control: public, max-age=2592000, immutable` — URL uniquely identifies a run, bytes never change |
| Historical runs on R2 | — | Current + 2 prior retained |
| Basemap | `map-assets.open-meteo.com` styles | OpenFreeMap (positron / dark) + heavy filter |
| UI chrome | Dark/Help/Settings/Clipping/Hillshade + full variable picker | OverlayPills (wind/gust/rain) + ModelPills + TimezoneSelector only |
| Timezone | Browser local or UTC | IANA dropdown + RN override |
| Model-run picker | Dropdown lets user choose any run + lock | Removed — always latest |
| Wind color scale | Library default | Surfr palette via hard resolver override |
| Arrow density | Widths 1.5–2.8, alpha 0.2–0.7 | Widths 0.7–1.3, alpha 0.15–0.4 |
| Embed integration | None | `?embed=1` → full RN postMessage bridge + diagnostic `t`-stamped events |
| SSR | Default (enabled) | Disabled |
| Surfr spots | n/a | Optional cyan dots + labels above zoom 8 |
| ADR | n/a | `docs/adr/0001-caching-architecture.md` |
