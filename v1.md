## 1. Tile caching & CDN architecture (brand-new)

The largest addition. A full 4-tier Cloudflare cache stack was built on top of
the upstream client so that all `.om` / `latest.json` / `meta.json` traffic
flows through `maps.thesurfr.app/tiles/*` instead of Open-Meteo's origin
directly. See `docs/adr/0001-caching-architecture.md` for the full decision
record.

### 1.1 Cloudflare Pages Function proxy — `functions/`

| File | New | Purpose |
|---|---|---|
| `functions/tiles/[[path]].ts` | +492 | The catch-all tile proxy. Reads R2 for `.om` and JSON indexes, falls back to origin for `.om` only, strips/fills client-side run paths, exposes CORS + Range support and `X-Surfr-Cache-Status` header. JSON indexes serve from R2 only (503 when cold). |
| `functions/tiles/_admin.ts` | +312 | `/tiles/_admin` HTML dashboard. Per-domain status pills (OK/STALE/COLD/UNKNOWN), R2 vs upstream `latest.json` comparison, file count + MB, last cron tick metadata. |
| `functions/tiles/_warmer-trigger.ts` | +81 | HTTP entry-point for warming. `?wait=1` blocks, default fire-and-forget via `waitUntil`. Validates `?domain=` against the known list. |
| `functions/lib/warmer.ts` | +424 | Core warm pipeline: fetch upstream `latest.json` → diff against R2 → fetch `meta.json` → warm `.om` files concurrency=4, +72h horizon cap → atomic swap `meta.json` then `latest.json` → prune old-run `.om` files. Emits per-run summary to `_warmer/last-run.json`. |
| `functions/lib/domains.ts` | +25 | Canonical list of 14 warmed tile domains (metno_nordic_pp, arome_france_hd, icon_d2, knmi_nl, ukv_2km, arome_france0025, cmc_gem_hrdps, hrrr_conus, knmi_europe, icon_eu, ecmwf_ifs025, icon, ncep_gfs013, ncep_gfs025). |
| `wrangler.toml` | +7 | Pages project config; binds `TILE_CACHE` R2 bucket (`surfr-tile-cache`). |

Key behaviours:
- **Run-path stripping:** client URLs omit `YYYY/MM/DD/HHmmZ`; the proxy fills
  it in from R2's canonical `latest.json`. Eliminates stale-URL races after a
  new run publishes.
- **Atomic run swap:** `meta.json` is written first, `latest.json` second, so
  readers never see a reference time before its manifest is in place.
- **Horizon cap:** 72 h max from reference_time — saves R2 writes/storage for
  rarely-viewed long-range hours.
- **In-isolate memory cache** for `latest.json` (30 s TTL) so `.om` hot paths
  don't re-read R2 metadata on every request.
- **Debug endpoint:** `/tiles/_debug/cache?prefix=…` returns JSON inventory +
  per-domain totals + last cron summary.

### 1.2 Cron Worker — `worker-cron/` (brand-new)

Separate tiny Cloudflare Worker whose only job is to fire the warmer every
5 min and purge the CF edge cache when a run swaps.

| File | New | Purpose |
|---|---|---|
| `worker-cron/src/index.ts` | +203 | `scheduled()` handler that sequentially hits `/tiles/_warmer-trigger?domain=…&wait=1` for each of the 14 domains, with a 1.5s pause between domains. Per-domain sharding avoids the "metno succeeded but everything else got 'no-upstream'" tail we saw with `warmAll`. `GET /force?domain=X` manual recovery endpoint. |
| `worker-cron/src/purge.ts` | +98 | Purges CF edge cache for stripped client URLs on a fresh `warmed` result. Chunks of 30 URLs (free-plan limit). `PURGE_HORIZON_HOURS = 72`. Deliberately does **not** re-warm — CF purge is global but re-warm only populates the calling PoP. |
| `worker-cron/wrangler.toml` | +18 | Cron trigger `*/5 * * * *`, `CF_ZONE_ID` plain var, `CF_PURGE_TOKEN` expected as secret. |
| `worker-cron/package.json` | +14 | `surfr-tile-warmer-cron`. |
| `worker-cron/tsconfig.json` | +14 | ESNext + workers-types. |
| `worker-cron/package-lock.json` | +1,589 | Lockfile (wrangler + workers-types). |

## 3. Embed mode (React Native WebView integration)

`src/lib/rn-bridge.ts` (+352, new) — Bidirectional postMessage bridge active
only when `?embed=1` is in the URL. Out-messages: `ready`, `moveend`,
`availableTimestamps`, `availableVariables`, `timestampChanged`,
`forecastLocationSet`, `referenceTime`, `tileFetch`, `storageEstimate`,
`setTimeReceived`, `setVariableReceived`, `setDomainReceived`,
`mapDataLoading`, `mapIdle`. In-messages: `setCenter`, `setForecastLocation`,
`setVariable`, `setDomain`, `setTime`, `setTzOffsetSeconds`,
`setSpotsConfig`. Every out-message stamps `t = performance.now()` so the
host can reconstruct a diagnostic timeline.

`src/routes/+page.svelte` (−32 / +148) —
- Reads URL params on mount: `clearCache`, `theme`, `opacity`,
  `tz_offset_seconds`, `spots_endpoint`, `spots_token`.
- Runs an `isEmbedMode()` check; if embed, hides the map popup and MapLibre's
  default attribution control.
- Installs `rnBridgeCleanup` in `onDestroy`.
- Swaps the upstream chrome (Dark/Help/Settings/Clipping/Hillshade buttons,
  `Scale`, full `VariableSelection`, `ClippingPanel`, `HelpDialog`, `Dropzone`,
  `Settings`) for a stripped-down standalone UI: `OverlayPills` +
  `ModelPills` + `TimezoneSelector` + `TimeSelector` + `KeyboardHandler`.
- Adds boot-time console audit of the wired color scales.

`src/lib/map-controls.ts` (−8 / +110) —
- `setMapControlSettings({ embed })` — early-return before adding
  MapLibre's `NavigationControl` / `GeolocateControl` / `GlobeControl` when
  embedded.
- `getStyle()` switched from `map-assets.open-meteo.com/styles/minimal-planet-maps*`
  to `tiles.openfreemap.org/styles/{positron|dark}` (CORS-open).
- Post-load basemap filter: hides transportation/buildings/landcover/landuse/
  waterway/aeroway/park source-layers; hides sub-national admin + micro-place
  labels; forces country borders white (filter excludes maritime boundaries);
  white flat place labels; adds a dedicated `surfr_coastline` line layer
  tracing ocean-class water polygons at interpolated width 0.4–1.4px.
- Wheel/touch zoom rate set to `1/85` (was `1/85` added late for both paths).

## 5. Display-timezone rework

`src/lib/stores/preferences.ts` (+23) — Adds `displayTimezone` (persisted IANA
string, defaults to browser TZ) and `displayTzOffsetSeconds` (writable in
seconds, set by RN embed bridge or derived from IANA name).

`src/lib/time-format.ts` (+110) — New helpers:
- `startOfUTCDay` / `withUTCTime` — pure UTC variants.
- `toShifted` / `fromShifted` — the "shift by offset" trick so UTC getters
  read target-TZ values while Date objects stay UTC internally.
- `startOfDisplayDay`, `withDisplayTime`, `formatDisplayDate/Time/DateTime`,
  `getDisplayDate/Month`.
- `getIanaOffsetSeconds(tz, at)` — DST-aware, uses `Intl.DateTimeFormat.formatToParts`.

`src/lib/components/time/time-selector.svelte` (−100 / +65) — Replaces every
`formatLocalDate/Time` + `startOfLocalDay` + `withLocalTime` with the new
display-tz equivalents; removes the entire model-run dropdown + lock button
(R2 is now the single source of truth for "current run", so users can't pick
older runs); removes the `UTC±HH:MM` badge above the timeline (redundant
with `TimezoneSelector`).

## 6. Color scales & rendering

`src/lib/color-scales/surfr.ts` (+67, new) — Surfr wind palette. RGB anchors
from WindguruService.WIND_COLOR_ANCHORS pushed to max-saturation per band
(blue → cyan → green → lime → yellow → orange → red-orange → pink → magenta
→ purple). Alpha floor raised to 0.35 for readability over dark basemaps.
Linearly interpolated to 1-kt resolution (breakpoints in m/s).

`src/lib/stores/om-protocol-settings.ts` (−6 / +98) —
- Registers `surfrWindScale` against keys `wind`, `wind_speed_10m`,
  `wind_gusts_10m`, `wind_u_component_10m`, `wind_v_component_10m`.
- Wraps `defaultResolveRequest` with a hard override: any variable matching
  `/^wind_(speed|gusts|u_component|v_component)(_|$)/` gets its `colorScale`
  and `intervals` force-replaced with `surfrWindScale` (both light + dark
  mode resolved). Belt-and-braces on top of the `colorScales` map after
  observing the library's step lookup repeatedly bypassing the registered
  override.
- `postReadCallback`: introduces a single-flight `AbortController` for
  prefetching adjacent-hour `.om` files so a lingering prefetch can't block
  the user's next scrub click (previous upstream behavior would stall HTTP/2
  streams for ~10s). Now `setToOmFile` + `prefetchVariable('not_a_real_variable')`
  run in an async IIFE, `AbortError` swallowed.

`src/lib/layers.ts` (−20 / +77) —
- Raster opacity: drop the upstream `-10pp` dark-mode reduction (raw pref
  in both modes) — our stripped basemap would otherwise look washed out.
- Arrow color: max alpha capped at 0.4 (was 0.7) so arrows hint direction
  without overpowering the raster.
- Arrow width: roughly halved across all thresholds (1.3 / 1.1 / 1.0 / 0.9 /
  0.8 / 0.7, was 2.8 / 2.2 / 2.0 / 1.8 / 1.6 / 1.5).
- `resolveBeforeLayer(map, desired)` — resilient layer-id lookup: falls
  back to first `boundary*` / `symbol` layer if the requested id isn't in
  the active style (OpenFreeMap uses different ids than the old
  open-meteo basemap).
- `changeOMfileURL` also updates `rasterManager.setBeforeLayer` (upstream
  only did it for vector).
- `onSlowLoad`: forward the slow-load toast to the RN host via
  `postMessage({ type: 'slowLoadWarning' })`.

`src/lib/constants.ts` (−5 / +12) —
- `BEFORE_LAYER_RASTER` / `BEFORE_LAYER_VECTOR` / `BEFORE_LAYER_VECTOR_WATER_CLIP`
  all switched to `waterway_line_label` (the id that exists in OpenFreeMap).
- `DEFAULT_CACHE_BLOCK_SIZE_KB` bumped 64 → 192 (3× fewer range requests per
  viewport render, less HTTP/2 multiplexing contention).

## 7. Surfr spots layer

`src/lib/surfr-spots.ts` (+160, new) — Renders Surfr user's saved surf/kite
spots as cyan circles + white labels above zoom 8. Fetches from
`${endpoint}/spots/bbox?…&limit=100` with bearer token, debounced 500 ms on
moveend. Endpoint + token come from URL params (`spots_endpoint`,
`spots_token`) or RN bridge (`setSpotsConfig`). Cancels in-flight bbox fetch
via `AbortController` on every map move.

## Summary of behavioural differences vs upstream

| Concern | Upstream | Surfr fork |
|---|---|---|
| Tile origin | Open-Meteo's CDN direct | CF Pages Function proxy + R2 + 5-min cron warmer |
| Client URL shape | Full `YYYY/MM/DD/HHmmZ/<validTime>.om` | Stripped; server fills run-path from R2 |
| Basemap | `map-assets.open-meteo.com` styles | OpenFreeMap (positron / dark) + heavy filter |
| UI chrome | Dark/Help/Settings/Clipping/Hillshade + full variable picker | OverlayPills (wind/gust/rain) + ModelPills + TimezoneSelector only |
| Timezone | Browser local or UTC | IANA dropdown + RN override |
| Model-run picker | Dropdown lets user choose any run + lock | Removed — R2 is single source of truth |
| Wind color scale | Library default | Surfr palette via hard resolver override |
| Arrow density | Widths 1.5–2.8, alpha 0.2–0.7 | Widths 0.7–1.3, alpha 0.15–0.4 |
| Embed integration | None | `?embed=1` → full RN postMessage bridge + diagnostic `t`-stamped events |
| SSR | Default (enabled) | Disabled |
| Surfr spots | n/a | Optional cyan dots + labels above zoom 8 |
| ADR | n/a | `docs/adr/0001-caching-architecture.md` |
